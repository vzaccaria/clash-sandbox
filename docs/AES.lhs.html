Introduction

The main program contains the top entity that will be synthesized by the Clash compiler.

module AES where

import CLaSH.Prelude
import Control.Monad.Trans.State.Lazy

The Types import defines, among other things, the AES state and the type of a state processor, i.e.:

  • the state is an vector of 16 bytes.
  • a state processor is just a function that takes a state and produces the next state.
import Types
import ShiftRows
import SubBytes

The AES Round

In the actual implementation of the AES, the round can be decomposed into the following pipeline stages:

  1. SubBytes
  2. ShiftRows
  3. MixColumns
  4. AddRoundKey

Here the round is defined by a sequence of state processing steps that are combined into a pipeline. We use a bind to connect two state processors:

round:: AESStateProcessor
round = _subBytes `bind` _shiftRows

To create the AES FSM we extract the transfer function from input state and output state by using an accessor to the state:

nextAESState :: AESState -> AESState
nextAESState  = execState AES.round

The synthesized FSM

The final FSM is just something that takes an initial state and no inputs and produces an output equal to the state.

aesMealy:: AESState -> () -> (AESState, AESState)
aesMealy s () = (s', s')
  where
    s' = nextAESState s

The TopEntity is going to be a synthesized Mealy machine with aesMealy as the transition function.


topEntity :: Signal () -> Signal AESState
topEntity = mealy aesMealy aesInitState
